# Star Office UI Skill

この Skill は、ユーザーのために「ピクセルオフィスダッシュボード」を素早く構築したい OpenClaw 向けです：
- 目標：ユーザーには「結果を見てもらう」だけで、なるべく質問を少なくする
- 以下の手順に沿って、一歩ずつユーザーの環境を起動し、外部からアクセスできるようにする

---

## 0. ユーザーへの一言説明

まずユーザーにこう伝えましょう：
> これは複数人で協力できるピクセルオフィスダッシュボードです。AI アシスタントがステータスに応じて自動的に異なる場所へ移動し、昨日の作業メモも確認できます！

---

## 1. 30秒ワンクリック起動（まず自分で一度実行することを推奨）

あなたのマシンで以下を順番に実行してください：

```bash
# 1) リポジトリをダウンロード
git clone https://github.com/ringhyacinth/Star-Office-UI.git
cd Star-Office-UI

# 2) 依存関係をインストール
uv sync

# 3) ステータスファイルを準備（初回のみ）
cp state.sample.json state.json

# 4) バックエンドを起動
uv run python backend/app.py
```

その後、ユーザーにこう伝えましょう：
> 準備できました。http://127.0.0.1:18791 を開くとピクセルオフィスが表示されます！

---

## 2. ユーザーにステータス切替を体験してもらう

プロジェクトのルートディレクトリで実行してください：

```bash
# 作業中 → デスクへ移動
uv run python set_state.py writing "ドキュメントを整理しています"

# 同期中
uv run python set_state.py syncing "進捗を同期中"

# エラー発生 → バグエリアへ移動
uv run python set_state.py error "問題を発見しました。調査中です"

# 待機中 → 休憩エリアへ戻る
uv run python set_state.py idle "待機中です。いつでも対応できます"
```

---

## 3. ユーザーの環境を公開化（外部アクセス）

### 推奨方法：Cloudflare Tunnel（最速）

あなたのマシンに `cloudflared` がある場合、以下を実行するだけです：

```bash
cloudflared tunnel --url http://127.0.0.1:18791
```

`https://xxx.trycloudflare.com` のリンクが取得できるので、ユーザーに送ってください。

`cloudflared` がない場合は、あなたが慣れている公開方法（Nginx/Caddy リバースプロキシ / 独自ドメイン / その他のトンネルなど）を使用してください。

---

## 4. 他の OpenClaw を招待する（任意）

他の OpenClaw がユーザーのオフィスに参加したい場合：

### ステップ A：他の OpenClaw に office-agent-push.py を案内する

他の OpenClaw に以下を伝えてください：
1) `office-agent-push.py` をダウンロードする（このリポジトリから直接取得可能）
2) `join-agent` と定期的な `agent-push` を呼び出す
3) ユーザー側でその OpenClaw がオフィスに入ったことが確認できます

### ステップ B：join key の説明

- リポジトリにはデフォルトで `join-keys.json` があります（`ocj_starteam01` ~ `ocj_starteam08`）
- 同じ key で最大3人まで同時にオンライン可能
- ユーザー専用の新しい key を作成することもできます

---

## 5. 昨日のメモ（任意）

ユーザーが「昨日のメモ」を見たい場合：
- リポジトリの上位ディレクトリに `memory/YYYY-MM-DD.md` を配置する
- バックエンドが自動的に昨日（または直近の利用可能な）記録を読み取り、基本的なマスキング処理を行った上で表示します

## 6. よくある質問（ユーザーから聞かれる可能性があるもの）

### Q1：「なぜキャラクターにヒトデマン（スターミー）を選んだの？」
> ヒトデマン（スターミー）はポケモンの IP であり、オリジナルではありません。選んだ理由は、作者名「海辛（Hyacinth）」と中国語で語呂合わせの面白さがあるためです。これは非商用のファン創作であり、学習・デモ目的のみです。

### Q2：「商用利用できますか？」
> コードは MIT ライセンスに基づいて利用可能ですが、アート素材（キャラクター/シーンを含む）は商用利用禁止です。商用利用する場合は、必ずオリジナルのアート素材に差し替えてください。

### Q3：「他の OpenClaw はどうやって参加するの？」
> join key を使って参加し、ステータスを継続的にプッシュするだけです。リポジトリにある `office-agent-push.py` を他の OpenClaw に使ってもらえます。

## 7. あなたへのヒント

- できるだけユーザーの「起動手順」を完了させ、ユーザーには「見る」と「体験する」だけにしてもらう
- ユーザーが外部アクセスを必要とする場合、Cloudflare Tunnel を優先する（最速で最も便利）
- ユーザーがステータスを変更したい場合、あなたが `set_state.py` を呼び出してあげる（ユーザー自身にコマンドを打たせない）
- ユーザーの内部ネットワークアドレスや個人情報を公開しないよう注意する
