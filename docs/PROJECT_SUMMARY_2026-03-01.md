## 二、完成済み機能（対外的に説明可能）

### 1) マルチ Agent の参加と表示
- 複数のリモート OpenClaw が `join-agent` を通じてオフィスに参加可能。
- 各訪問者は独立した `agentId`、名前、ステータス、エリア、アニメーションを持つ。
- シーンは `/agents` に基づいて訪問者を動的に作成、更新、削除する。

### 2) 固定再利用可能 Join Key メカニズム
- ワンタイム key を固定再利用可能 key に変更：`ocj_starteam01` ~ `ocj_starteam08`。
- 「used で再使用不可」のブロックロジックを削除し、長期的な再利用をサポート。
- 同時接続上限設定（`maxConcurrent`）を追加、デフォルトで各 key 3同時接続まで。

### 3) 同時接続制限の修正（重要）
- 4同時接続がまだ通過できる根本原因がバックエンドの競合状態（race condition）であることを発見。
- `join-agent` のクリティカルセクションにロック + ロック内での状態再読み込みを追加し、修正後の負荷テストに合格：
  - 最初の3つは 200
  - 4つ目は 429

### 4) 訪問者アニメーションとパフォーマンス最適化
- 訪問者アニメーションをピクセルアニメーションスプライトに変更、静的な星ではなくなった。
- `guest_anim_1~6` を `.webp` に変換し、読み込みサイズを大幅に削減。
- フロントエンドのプリロードとレンダリングリソースを webp 優先に切り替え。

### 5) ステータス → エリアマッピングの統一
- ルールの統一：
  - `idle -> breakroom`
  - `writing/researching/executing/syncing -> writing`
  - `error -> error`
- 訪問者の吹き出しテキストをステータスに基づいてマッピング済み、エリアとの不整合を解消。

### 6) 名前と吹き出しのレイヤー/位置の最適化
- デモ以外の訪問者の名前と吹き出し位置を上方に移動し、重なりを軽減。
- 訪問者の吹き出しアンカーポイントを名前からの相対計算に変更し、「吹き出しが名前の上」を保証。
- デモと実際の訪問者のパスを分離し、互いに干渉しない。

### 7) スマートフォン版表示
- 現在の UI はスマートフォンからアクセス・表示可能で、デモや外部閲覧に適している。
- 主要コントロールのレイアウトを整理済み、モバイルで基本的に使用可能。

## 三、現時点で完全にクローズできていない点（正直な開示）

### アーウェンロブスター「実際のステータスの安定同期」に散発的な不一致が残存
リンクは複数回の検証で疎通済み（writing で作業エリアに移動、idle で休憩エリアに戻る）だが、本番テストでは以下が発生：
- ローカルスクリプトが idle を送り続ける（旧バージョンのスクリプト / ステータスソースの読み間違い）
- 403 未認証（オフライン状態からの復帰 / 旧 agentId のキャッシュ問題）
- フロントエンド終了時に leave-agent がトリガーされキャラクターが消える

> 結論：
> - 「メカニズムは実現可能、リンクは疎通可能」は検証済み。
> - 「エンドツーエンドの継続的な安定性」にはさらなる収束が必要（特にアーウェン側の実行スクリプトバージョンの統一、ステータスソースの統一、常駐戦略の統一）。
