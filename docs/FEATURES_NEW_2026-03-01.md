# Star Office UI — 新機能説明（本フェーズ）

## 1. マルチロブスター訪問者システム
- 複数のリモート OpenClaw が同一オフィスに同時参加可能。
- 訪問者ごとに独立したアバター、名前、ステータス、エリア、吹き出しをサポート。
- 動的なオンライン・オフラインとリアルタイム更新に対応。

## 2. Join Key メカニズムのアップグレード
- 「ワンタイム key」から「固定再利用可能 key」にアップグレード。
- デフォルト key：`ocj_starteam01` ~ `ocj_starteam08`。
- セキュリティ制御を維持：各 key の同時接続上限 `maxConcurrent`（デフォルト 3）。

## 3. 同時接続制御（競合状態を修正済み）
- 同時 join の競合状態（race condition）を修正。
- 同一 key の4番目の同時 join は正しく拒否される（HTTP 429）。

## 4. 訪問者ステータスマッピングとエリアレンダリング
- `idle -> breakroom`
- `writing/researching/executing/syncing -> writing`
- `error -> error`
- 訪問者の吹き出しテキストがステータスと同期し、ずれが解消。

## 5. 訪問者アニメーションとリソース最適化
- 訪問者を静止画からアニメーションスプライト（ピクセルアート風）にアップグレード。
- `guest_anim_1~6` の webp バージョンを提供し、読み込みサイズを削減。

## 6. 名前と吹き出し表示の最適化
- デモ以外の訪問者の名前と吹き出し位置を上方に移動し、キャラクターとの重なりを回避。
- 吹き出しのアンカーポイントを名前基準の配置に変更し、「吹き出しが名前の上」を保証。

## 7. モバイル表示
- ページはスマートフォンから直接アクセス・表示可能。
- レイアウトは基本的なモバイル対応済みで、デモシーンに対応。

## 8. リモートプッシュスクリプト連携の改善
- ステータスファイルからの読み取りとオフィスへのステータスプッシュをサポート。
- ステータスソース診断ログを追加（「なぜずっと idle なのか」の特定用）。
- AGENT_NAME 環境変数の上書きタイミング問題を修正。
