# Star Office UI — 機能説明（概要）

Star Office UI は「ピクセルオフィス」の可視化インターフェースで、AI アシスタント / 複数の OpenClaw 訪問者のステータスを、ウェブページ（スマートフォン含む）で閲覧できるオフィスシーンとしてレンダリングします。

## 表示内容
- ピクセルオフィスの背景（俯瞰図）
- キャラクター（Star + 訪問者）がステータスに応じて異なるエリアに移動
- 名前と吹き出し（bubble）で現在のステータス/考え（カスタムマッピング可能）を表示
- スマートフォンでも表示可能（作品展示/配信/対外デモに最適）

## コア機能

### 1) 単一 Agent（ローカル Star）のステータスレンダリング
- バックエンドが `state.json` を読み取り `GET /status` を提供
- フロントエンドが `/status` をポーリングし、`state` に基づいて Star の所在エリアをレンダリング
- `set_state.py` で素早くステータスを切り替え可能

### 2) マルチ訪問者（マルチロブスター）のオフィス参加
- 訪問者は `POST /join-agent` で参加し、`agentId` を取得
- 訪問者は `POST /agent-push` で自身のステータスを継続的にプッシュ
- フロントエンドは `GET /agents` で訪問者リストを取得しレンダリング

### 3) Join Key（接続キー）メカニズム
- 固定再利用可能な join key をサポート（例：`ocj_starteam01~08`）
- 各 key の同時オンライン上限をサポート（デフォルト 3）
- 「誰がオフィスに入れるか」と「同一 key で同時に何匹のロブスターが入れるか」を制御可能

### 4) ステータス → エリアマッピング（統一ロジック）
- idle → breakroom（休憩エリア）
- writing / researching / executing / syncing → writing（作業エリア）
- error → error（障害エリア）

### 5) 訪問者アニメーションとパフォーマンス最適化
- 訪問者キャラクターはアニメーションスプライトを使用
- WebP リソースをサポート（より小さいサイズ、より速い読み込み）

### 6) 名前/吹き出しが重ならないレイアウト
- 実際の訪問者とデモ訪問者のロジックを分離
- デモ以外の訪問者の名前と吹き出しを全体的に上方に移動
- 吹き出しを名前の上にアンカリングし、名前に重なることを回避

### 7) デモモード（オプション）
- `?demo=1` の場合のみデモ訪問者を表示（デフォルトでは非表示）
- デモと実際の訪問者は互いに影響しない

## 主要 API（バックエンド）
- `GET /`：フロントエンドページ
- `GET /status`：単一 agent のステータス（旧バージョンとの互換性）
- `GET /agents`：マルチ agent リスト（訪問者レンダリング用）
- `POST /join-agent`：訪問者の参加
- `POST /agent-push`：訪問者のステータスプッシュ
- `POST /leave-agent`：訪問者の退出
- `GET /health`：ヘルスチェック

## セキュリティとプライバシーに関する注意
- プライバシー情報を `detail` に書き込まないこと（レンダリング/取得される可能性があるため）
- オープンソース化前に必ず整理すること：ログ、実行状態ファイル、join keys、トンネル出力など

## 美術アセット使用に関する声明（必須）
- コードはオープンソース可能だが、美術素材（背景、キャラクター、アニメーションなど）の著作権は原作者/スタジオに帰属。
- 美術アセットは学習とデモ用途のみ、**商用利用禁止**。
