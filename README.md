# Star Office UI

マルチエージェント協働のためのピクセルオフィスダッシュボード：AIアシスタント（OpenClaw / ロブスター）の作業状態をリアルタイムで可視化し、「誰が何をしているか、昨日何をしたか、今オンラインかどうか」をチームが直感的に把握できるようにします。

![Star Office UI プレビュー](docs/screenshots/office-preview-20260301.jpg)

---

## このプロジェクトとは？（一言で）

Star Office UI は「マルチユーザー協働ステータスダッシュボード」です。イメージとしては：
> リアルタイムに更新される「ピクセルオフィスダッシュボード」：あなたの AI アシスタント（および招待した他のエージェント）がステータスに応じて自動的にオフィス内の異なるエリア（休憩所 / デスク / バグエリア）に移動し、昨日の作業メモも確認できます。

## 30秒クイックスタート（まずはここから）

```bash
# 1) リポジトリをクローン
git clone https://github.com/ringhyacinth/Star-Office-UI.git
cd Star-Office-UI

# 2) 依存関係をインストール
uv sync

# 3) 状態ファイルを初期化（初回のみ）
cp state.sample.json state.json

# 4) バックエンドを起動
uv run python backend/app.py
```

ブラウザで開く：**http://127.0.0.1:18791**

ステータスを切り替えてみましょう（プロジェクトルートで実行）：
```bash
uv run python set_state.py writing "ドキュメント整理中"
uv run python set_state.py syncing "進捗同期中"
uv run python set_state.py error "問題発見、調査中"
uv run python set_state.py idle "待機中"
```

## 1. このプロジェクトで実現していること

Star Office UI が現在提供する機能：

1. **ロブスターの作業ステータスを可視化**
   - ステータス：`idle`（待機）、`writing`（作業中）、`researching`（調査中）、`executing`（実行中）、`syncing`（同期中）、`error`（バグ報告）
   - ステータスはオフィス内の異なるエリアにマッピングされ、アニメーションや吹き出しで表示されます。

2. **「昨日のメモ」マイクロサマリー**
   - フロントエンドに「昨日のメモ」カードを表示。
   - バックエンドが `memory/*.md` から昨日（または直近の利用可能な）記録を読み取り、基本的な匿名化処理を行って表示します。

3. **他のゲストをオフィスに招待可能（機能は継続的に改善中）**
   - join key で参加。
   - ゲストは自分のステータスをオフィスダッシュボードに継続的にプッシュできます。
   - 現在利用可能ですが、インタラクションとオンボーディング体験は引き続き最適化中です。

4. **モバイル対応済み**
   - モバイル端末から直接アクセスしてステータスを確認できます（外出先での素早い確認に最適）。

5. **柔軟な公開アクセス方法**
   - Skill はデフォルトで Cloudflare Tunnel による素早い公開アクセスを推奨。
   - 独自の公開ドメインやリバースプロキシ設定も利用可能です。

## 4. 主要 API

- `GET /health`：ヘルスチェック
- `GET /status`：メインエージェントのステータス
- `POST /set_state`：メインエージェントのステータスを設定
- `GET /agents`：マルチエージェント一覧を取得
- `POST /join-agent`：ゲストが参加
- `POST /agent-push`：ゲストがステータスをプッシュ
- `POST /leave-agent`：ゲストが退出
- `GET /yesterday-memo`：昨日のメモ
